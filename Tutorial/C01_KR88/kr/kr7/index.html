Chapter 7 - Input and Output

Lower 2.   The program kr153.c converts its input to lower case.

Formatted Output.   The program kr154.c illustrates a variety of specifications in printing "hello, world".

Variable-length Argument List.   The program kr156.c reads a character, integer, float and word and prints them out with minprintf(s,...), a minimal printf with a variable-length argument list.

Rudimentary Calculator 2.   The program kr158.c reads long floats in input, adds them up, and prints the running sum after each input.

scanf:   A simple scanf example.
The program kr159.c reads a date form 25 dec 1988 and prints it, and then reads a date of form mm/dd/yy and prints it.

sscanf:  A simple sscanf example.
The program kr159_2.c reads a date in input, one per line, and checks with sscanf whether the input is a valid date form (accepted forms: 25 dec 1988, mm/dd/yy), and prints them. Note that this program is not fully error free, as for example the input 3/3/3 2 will be interpreted as a valid form of the first form: 3 /3/3 2

File Access:  Concatenation of n files.
The program kr162.c reads n files in input given as arguments in the command line, concatenates them and prints the result to the standard output. If no arguments are given in the command line, the standard input is readed and copied line per line to the standard output.
Synopsis:  kr162   [ FILE ]

Error Handling - sdterr and exit.   The program kr163.c is a revised version of kr162.c, using stderr for error treatment.

char *fgets(*s, n, *f),   int fputs(*s, *f):   fgets reads the next n-1 characters into the string s, stopping if a newline is encountered and fputs writes the string s on stream f. Note that fgets returns NULL on end of file or error.
The program kr165.c copies an input file to an output file.
Synopsis:  kr165   inputFILE   outputFILE

int getline(*s, n)  (version 3):   getline reads the next n-1 characters into the string s, stopping if a newline is encountered. Note that getline returns, unlike fgets, the length of the readed string s, or zero on end of file.
The test program kr165_2_t.c reads a file from the standard input, chunk by chunk of maximum length n-1 given as argument in the command line, and prints them to the standard output with their length.
Synopsis:   kr165   n

int getc(*f), int ungetc(c, *f)  (version 2):   A simple buffer management, similar to the one studied in chapter 4, see kr079.c: getc(f) returns the next character of stream f (or EOF if end of file or error ocurs) and ungetc(c,f) pushes c back onto stream f, where it will be returned on the next read (and returns the character pushed back, or EOF for error).
The test program kr166_t.c is similar to kr079_t.c and interactively gets and ungets characters from an input file given as argument in the command line. Note that some elementary error treatment is provided by getc and ungetc, but not by the test program itself. The text file kr166_i.txt can be used as input file.
Synopsis:  kr166_t   inputFILE

system(char *s):   Execute the command contained in the string s.
The test program kr167_t.c reads UNIX command from the standard input and executes them. Hint: Try out simple ones as date, but also more complex ones as egrep -l MAX *.c.

malloc, calloc, free:   Storage Management.
The program kr167_2_t.c allows to interactively test the three primitives malloc, calloc and free. The text file kr167_2_i.txt can be used as a batch input file for a first test, and contains also a little explanation.

int rand(),  void srand(seed)  (version 2):   Random Number Generator.
The test program kr168_t.c reads two integers s and n, and generates and prints a series of n pseudo-random numbers between 0 and 1 with seed s.

Reference: Brian W. Kernighan, Dennis M. Ritchie: The C Programming Language, 2nd Edition, Prentice Hall 1988
